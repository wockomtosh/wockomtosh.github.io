Boss AI

One of my tasks in Deep Sea Defender was the implementation of the boss AI.

Behavior Tree

This is the main portion of the boss behavior tree. When the player enters the boss area, the boss alternates between following the player and attacking. The boss has 3 attacks that it can choose from.

Attack Sequence

When the attack cooldown is complete, the boss runs a task to choose its attack. This task checks whether or not the game is using Twitch integration. If Twitch integration is active, the boss will look for the latest voting result from Twitch and use that to select an attack. Otherwise, it will select a random attack.

Individual Attacks

Each attack has three parts. This was done to let other team members more easily add their animations or effects to the attack in the correct place. From a programming perspective, the "begin" task sets up any values or prerequisites needed for the attack, the "execute" task actually performs the attack, and the "end" task resets values. In the case of the charge attack, the begin task increases the movement speed of the boss and calculates a location in front of the player to move to. Other team members played sounds and effects in this task as well. The execute task contains a move to function and only proceeds after the movement is finished. The end task resets the movement speed.

Health System

Another of my tasks in the game was creating the health system.
The enemies and the player blueprints implement a blueprint interface for damage and death. The interface has functions for taking damage, dealing damage, and death.
